# Мониторинг цен: Москва / СПб → Венеция (сентябрь, ₽ и €, обновление каждые 8 ч)

Скрипт раз в 8 часов собирает цены на вылет **в сентябре** (весь месяц) из
**Москвы и Санкт-Петербурга** в Венецию (VCE) и Тревизо (TSF), с фокусом на
пересадках **через Ереван, Турцию и Белград**, показывает цену **в рублях и
евро** (курс ЦБ РФ на день), копит историю в `data/history.csv` и пересобирает
`docs/index.html`. Всё бесплатно на GitHub Actions + GitHub Pages, сервер не нужен.

Итого 16 карточек: 2 города вылета × 2 пункта × 4 варианта маршрута
(любой + через Ереван / Турцию / Белград).

## Как считаются маршруты (важно, честно)
Для каждого пункта — три карточки:
- **Любой маршрут (мин.)** — самый дешёвый цельный билет с одной стыковкой
  (какой хаб — решает кеш API).
- **Через Ереван / Турцию / Белград** — сумма ДВУХ отдельных билетов
  (город вылета → хаб + хаб → пункт) за сентябрь. Это self-connect: часто
  дешевле, но это две разные брони и риск стыковки на тебе. Турция = минимум
  из Стамбула IST и Сабихи SAW; Белград — частый вариант Air Serbia.

Оговорки:
- Data API отдаёт цены **из кеша поисков за ~48 ч**, а не живые котировки.
  Это ориентир по тренду; финальную цену сверяй на aviasales.ru по ссылке.
- Месяц задаётся с точностью до сентября, поэтому сумма по хабу —
  индикативная оценка бюджета, а не гарантированная стыковка в конкретный день.
- Прямых рейсов Москва → Венеция сейчас нет.

## Запуск за 5 шагов
1. **Токен:** зарегистрируйся на travelpayouts.com → профиль → раздел «API token».
2. **Репозиторий:** создай на GitHub (можно приватный), залей `monitor.py`,
   `.github/workflows/monitor.yml`, `README.md`.
3. **Секрет:** Settings → Secrets and variables → Actions → New repository secret.
   Имя `TRAVELPAYOUTS_TOKEN`, значение — токен.
4. **Pages:** Settings → Pages → Source = «GitHub Actions».
5. **Первый запуск:** Actions → «fare-monitor» → «Run workflow».
   Дальше сам, каждые 8 ч (00:00 / 08:00 / 16:00 UTC).

Страница: `https://<логин>.github.io/<репозиторий>/`.

## Настройка (начало monitor.py)
- `DEPART_MONTH = "2026-09"` — месяц вылета (анализ по всему месяцу).
- `ORIGINS` — города вылета (сейчас Москва MOW и Санкт-Петербург LED).
- `DESTINATIONS` — пункты прилёта.
- `VIA_GROUPS` — хабы пересадки (Ереван / Турция / Белград); можно добавить свои.
- Периодичность — строка `cron` в `.github/workflows/monitor.yml`.

## Источники данных
- Цены — Travelpayouts (Aviasales) Data API.
- Курс евро — официальный ЦБ РФ, `https://www.cbr-xml-daily.ru/daily_json.js`
  (бесплатно, без ключа). Если недоступен — страница показывает только ₽.

## Локальный тест
```bash
export TRAVELPAYOUTS_TOKEN=твой_токен
python monitor.py         # затем открой docs/index.html
```
